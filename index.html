Software Design Principles

DRY

DRY is an acronym for Do not Repeat Yourself

Benefits of DRY

  • Reduced number of lines of code
  • Easier maintainability
  • Code reusability
  • Readability

Example of non DRY code

                
let chips = ['corn', 'pita', 'potato', 'tortilla'];

console.log(chips[0]);
console.log(chips[1]);
console.log(chips[2]);
console.log(chips[3]);

Example of DRY code

                
let chips = ['corn', 'pita', 'potato', 'tortilla'];
chips.forEach(item => {
  console.log(item);
});

Code reusability example

                
const logItems = (array) => {
  array.forEach(item => {
    console.log(item);
  });
}

Conclusion

Whenever you finish writing some code, you should always look back to see if there is any way you can DRY it up

Kiss Principle

KISS stands for Keep it Simple, Stupid

What is simple program

Simple program is program that:
  • has less features/parts in it
  • does not have unused/dead code
  • doesn't have many parts that are depended on other parts

How to write simple code

  • Ensure your function names translate to the purpose of that function
  • Ensure your function has a single responsibility.
  • Avoid global states and behaviors like as much as you can.
  • Delete unused code

Benefits

  • Maintainability
  • Readability
  • Easier to test

Conclusion

While writing your code always think if the program is easy to read and understand, is it easy to extend your code base. If not than try to refactor it.
